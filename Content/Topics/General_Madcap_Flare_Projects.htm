Create a new project

Always copy an existing folder to create a new project. This preserves the file mappings and pulls in the common stylistic and layout elements.

  1. Copy an existing project folder and rename it as required.
  2. Open the .flprj file in a text editor and change the SourceControlBound value to "false". This disables source control for the project but keeps the connection settings in the configuration.
  3. Rename the .flprj file as required.
  4. Open the project's properties dialog box and click Bind Project.
  5. Select Git as the source control provider.
  6. Check Remote Repository and enter the link to the remote repository.
  7. Enter your source control account Name and Email and click OK.

    Ensure that you have an SSH key saved for your source control account and that the owner has added this key for the repository.

  8. Open the project and click the project tab.
  9. Click Synchronize Files.

    Ensure that you are connected to SharePoint in SharePoint Explorer.

Commit project changes

MadCap Flare should be set up to check-in your changes by default when you open a project. Ensure that you also push your changes to the remote repository by clicking Push in the Source Control ribbon.

If you can find it, run the flare-git-sync.bat file to back up the remote repository to the Network drive.
