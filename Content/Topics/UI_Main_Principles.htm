Main design principles

General guidelines

  • Avoid using scroll-bars.
  • Avoid using virtual keyboards and other popovers.
  • Group similar operations into functional groups (Settings, Diagnostics, etc).
  • Don’t place items that belong to a group separately, unless they involve critical functionality.
  • Singular icons must be substantiated by text where space permits. This way you rely less on discoverability and facilitate user learning.

Always keep the user informed

At all times, the user must be able to:

  • access the dashboard;
  • access the main menu and navigation menu; and
  • see the status of the system, in particular any critical errors.

Use established mental models

Meet users’ expectations by adhering to established mental models as follows:

  • Use familiar UI elements (toggle switches, radio buttons, check boxes).
  • Don’t reinvent the wheel; use familiar interface paradigms (Material UI, generic mobile device OSs, and so on).

Use color sparingly and in a meaningful way

  • Use muted colors and reserve bright colors (reds, greens, yellows) only when you want to draw the user’s attention to an element.
