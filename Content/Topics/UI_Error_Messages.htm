Error messages

  • Use simple terms and plain language.
  • Write concisely, but precisely. Include just enough information to explain the problem and how to fix it.
  • Focus on the problem and not the user action that led to the problem.
  • Avoid using upper case text and (or) exclamation marks in the messages.
  • Do not report non-critical errors that are irrelevant to the user.
  • Disclose additional information progressively.
