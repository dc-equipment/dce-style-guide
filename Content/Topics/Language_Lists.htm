Lists

Use a list when you have three or more important points to reinforce or amplify. Use ordered lists for steps in a task instead of grouping them together in a single, monolithic block of paragraph text. Use unordered lists when the sequence of items is not important.

Follow these rules when you use lists:

  • Ensure that you make the purpose of the list clear in the heading.
  • Introduce a list with a complete sentence that ends with a colon, except in a horizontal list or a vertical list that is punctuated as a sentence.
  • Avoid creating lists that are more than three levels deep.
  • Never use a list for only one item and avoid two-item lists.
  • Use parallel construction for list items and write in the active voice. Refer to Watch your voice and tone for more information.
  • If using end punctuation, use it consistently for all items in the list.
  • Avoid including the number of steps in the introductory sentence.

Vertical list punctuated as a sentence

It is perfectly acceptable to complete a sentence begun in the introductory text with items in a vertical list. Use semicolons between the items and end the final item with a period. Each item begins with a lowercase letter, even if the list is a numbered list.

Example:

 

Take up any slack in the rope by:

  • using the Track Lockout Override function; or
  • using the manual wind-in function.

Horizontal (run-in) lists

Short, simple list are usually better run in, especially if the introductory text and the items in the list together form a sentence.

Follow these rules when you use run-in lists:

  • If the introductory sentence forms a grammatically complete sentence, use a colon to offset the list items from the sentence.
  • Use numbers or alphabets in parenthesis before each item.
  • Separate the items with commas, unless any of the items requires internal commas, in which case all items will usually need to be separated by a semicolon.

Example:

 

This presents certain obstacles to maintaining consistency across your projects because: (a) changes made to the layout in one project are not updated in the others; and (b) changes made in the same project by one writer are not updated in another writer's local copy.
